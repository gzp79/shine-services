Input System
Maps raw device input to locomotion state and gesture events for the avatar/camera systems.

Problem

The input system maps raw device events (pointer, keyboard, touch, wheel) to callbacks consumed by avatar/camera systems. A layered architecture separates concerns: raw inputs detect gestures, schemas compose raw inputs and apply conflict resolution, InputManager routes callbacks from the active schema, and external systems implement InputHandler to receive input events. This structure makes gesture detection, conflict rules, and schema switching independently testable.

Conflict / Exclusion Rules

Only block the gesture that does the same thing as the active key. Everything else stays active — WASD + right-mouse-rotate works, WASD + wheel zoom works. Rules are implemented in schemas by enabling/disabling raw inputs (e.g., WASD held → leftPointer.enabled = false).

Active keys drag-pan drag-rotate wheel zoom interact (long-press)
WASD held blocked — both move allowed ✓ allowed ✓ allowed ✓
Q/E held allowed ✓ blocked — both rotate allowed ✓ allowed ✓
R/F held allowed ✓ allowed ✓ blocked — both zoom allowed ✓
No keys allowed ✓ allowed ✓ allowed ✓ allowed ✓

Touch Conflict Rules

Single-finger and two-finger gestures are mutually exclusive. First gesture to start wins and remains active until all fingers are released.

Currently active tap/drag/long-press (single-finger) pinch (two-finger)
Single-finger gesture active active ✓ blocked — first to start wins
Two-finger pinch active blocked — first to start wins active ✓
No gesture allowed ✓ allowed ✓

Note: After two-finger pinch ends with one finger remaining, single-finger gestures stay blocked until all fingers are released.

Testing

Raw inputs are tested for gesture detection only. Schemas are tested for conflict resolution (enable/disable logic). InputManager is tested for schema tracking and callback routing.

Schema Conflict Rule Tests (DesktopSchema)

WASD blocks left pointer drag (but not long-press)

  • 1WASD held → left-drag blocked (callbacks ignored if wasdActive) block
  • 2Left-drag active → WASD pressed → WASD.enabled = false block
  • 3Left-drag ends → WASD.enabled = true state
  • 4WASD held → long-press still works (not blocked) allow
