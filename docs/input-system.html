Input System
Maps raw device input to locomotion state and gesture events for the avatar/camera systems.

Problem

The input system maps raw device events (pointer, keyboard, touch, wheel) to two outputs the avatar and camera systems consume: a locomotion state polled every frame, and one-shot gesture events. Conflict resolution between simultaneous inputs (e.g., WASD vs drag-pan) lives in a single place — InputManager — so the rules are testable in isolation and new input sources slot in behind the same InputHandler interface.

Conflict / Exclusion Rules

Only block the gesture that does the same thing as the active key. Everything else stays active — WASD + right-mouse-rotate works, WASD + wheel zoom works. Rules live entirely in InputManager.

Active keys drag-pan drag-rotate wheel zoom interact (long-press)
WASD held blocked — both move allowed ✓ allowed ✓ blocked
Q/E held allowed ✓ blocked — both rotate allowed ✓ allowed ✓
R/F held allowed ✓ allowed ✓ blocked — both zoom allowed ✓
No keys allowed ✓ allowed ✓ allowed ✓ allowed ✓

Testing

Controllers (DesktopController, TouchController) are tested for gesture detection only. Exclusion rules are tested exclusively at the InputManager boundary.
