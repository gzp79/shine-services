# Hex Mesh Viewer Design

Three.js client application that renders hex quad meshes generated by `PatchMesher` via wasm bindings, with interactive controls for subdivision, orientation, smoothing method, and parameters.

## Architecture Overview

```
shine-game (Rust)          client/ (TypeScript)
┌──────────────────┐       ┌──────────────────────┐
│ patch_mesher.rs   │       │ main.ts              │
│ wasm_api.rs ──────┼─wasm─►│ scene.ts             │
│  (cfg wasm32)     │  pkg  │ mesh-builder.ts      │
│                   │       │ controls.ts          │
└──────────────────┘       └──────────────────────┘
```

Two deliverables:
1. **Wasm bindings** — new `wasm_api.rs` in `shine-game` exposing mesh generation to JS
2. **Client app** — `client/` at repo root, TypeScript + pnpm + Vite + three.js

## 1. Wasm Bindings (`shine-game/src/wasm_api.rs`)

A `#[cfg(target_arch = "wasm32")]` module with a minimal wasm-bindgen API.

### API

```rust
#[wasm_bindgen]
pub struct WasmPatchMesh { /* internal */ }

#[wasm_bindgen]
impl WasmPatchMesh {
    /// Flat vertex positions [x, y, x, y, ...] (2 floats per vertex)
    pub fn vertices(&self) -> Vec<f32>;
    /// Flat quad indices [a, b, c, d, a, b, c, d, ...] (4 indices per quad)
    pub fn indices(&self) -> Vec<u32>;
}

/// Generate a hex quad mesh from a JSON config string.
#[wasm_bindgen]
pub fn generate_mesh(config_json: &str) -> WasmPatchMesh;
```

### Config JSON Schema

```json
{
  "subdivision": 3,
  "orientation": "Even",
  "seed": 42,
  "smoothing": {
    "method": "Lloyd",
    "iterations": 20,
    "strength": 0.4,
    "weight_min": 2.5,
    "weight_max": 15.5
  },
  "fix_quads": {
    "enabled": true,
    "min_quality": 0.15,
    "max_iterations": 50
  }
}
```

Smoothing method variants and their parameters:

| Method | Parameters |
|--------|-----------|
| None | (none) |
| Lloyd | iterations, strength, weight_min, weight_max |
| Noise | amplitude, frequency |
| Cotangent | iterations, strength |
| Spring | iterations, dt, spring_strength, shape_strength |
| Jitter | amplitude |

### RNG

A simple seeded xorshift32 implementing `StableRng`, built into `wasm_api.rs`. No external RNG dependency needed for the wasm target.

### Mesh Output

- `vertices()`: flat `Vec<f32>` of 2D positions `[x0, y0, x1, y1, ...]`
- `indices()`: flat `Vec<u32>` of quad corner indices `[a0, b0, c0, d0, a1, ...]` — 4 indices per quad, derived from `PatchDenseIndexer` iteration + `PatchCoord::quad_vertices` + `AxialDenseIndexer`

### Crate Changes

Add to `shine-game/Cargo.toml`:

```toml
[lib]
crate-type = ["cdylib", "rlib"]
```

Add `wasm_api` module to `lib.rs` (or equivalent entry point) behind `#[cfg(target_arch = "wasm32")]`.

## 2. Client Project (`client/`)

### Project Setup

- **Location**: `client/` at repo root
- **Package manager**: pnpm
- **Language**: TypeScript
- **Bundler**: Vite with `vite-plugin-wasm` + `vite-plugin-top-level-await`
- **3D**: three.js
- **UI**: lil-gui

### File Structure

```
client/
  package.json
  tsconfig.json
  vite.config.ts
  index.html
  src/
    main.ts             # entry: init wasm, scene, controls, wire them together
    scene.ts            # three.js scene, camera, lights, renderer, OrbitControls
    mesh-builder.ts     # wasm output → Three.js BufferGeometry + edge lines
    controls.ts         # lil-gui panel definition and callbacks
```

### Scene (`scene.ts`)

- **Camera**: Perspective camera positioned above the mesh looking down at the XZ plane (Y=up)
- **Controls**: OrbitControls for rotation, zoom, pan
- **Lighting**: Ambient light + one directional light from above-and-side
- **Renderer**: WebGLRenderer, antialias enabled, handles window resize

### Mesh Rendering (`mesh-builder.ts`)

Takes wasm output (flat vertices + quad indices) and builds:

1. **Filled geometry**: `BufferGeometry` with indexed triangles. Each quad (a,b,c,d) split into 2 triangles (a,b,c) and (a,c,d). `MeshStandardMaterial` with flat shading.
2. **Edge overlay**: `LineSegments` built explicitly from quad topology (4 edges per quad). Thin dark lines. Not using `EdgesGeometry` — all quad boundaries must be visible.
3. **Patch coloring**: Quads colored by patch index (3 patches = 3 subtle pastel colors) to visualize patch structure. Applied via vertex colors.
4. **Coordinate mapping**: 2D vertex `(x, y)` from wasm maps to Three.js `(x, 0, z)` — mesh lies on the XZ ground plane.

### Controls (`controls.ts`)

**Main controls** (always visible):

| Control | Type | Range | Default |
|---------|------|-------|---------|
| Subdivision | slider (int) | 0–5 | 3 |
| Orientation | dropdown | Even, Odd | Even |
| Smoothing | dropdown | None, Lloyd, Noise, Cotangent, Spring, Jitter | None |
| Seed | number | 0–999999 | 42 |

Auto-regenerates mesh on any change.

**Advanced folder** (collapsed, contents change based on smoothing):

- **Lloyd**: iterations (1–50, def 20), strength (0–1, def 0.4), weight min (0.5–20, def 2.5), weight max (1–30, def 15.5)
- **Noise**: amplitude (0–2, def 0.5), frequency (0.5–20, def 5.0)
- **Cotangent**: iterations (1–50, def 10), strength (0–1, def 0.5)
- **Spring**: iterations (1–200, def 50), dt (0.01–0.5, def 0.1), spring strength (0–2, def 0.3), shape strength (0–2, def 0.5)
- **Jitter**: amplitude (0–5, def 2.0)
- **Fix quads**: enabled (checkbox, def on), min quality (0.01–0.5, def 0.15), max iterations (1–200, def 50)

## 3. Build Pipeline

### Wasm Build

```bash
wasm-pack build crates/shine-game --target web --out-dir ../../client/pkg
```

- Outputs ES module to `client/pkg/`
- `client/pkg/` added to `.gitignore`

### Client Scripts (`package.json`)

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "wasm": "wasm-pack build ../../crates/shine-game --target web --out-dir ../../client/pkg",
    "wasm:dev": "pnpm wasm && pnpm dev"
  }
}
```

### Dev Workflow

1. `cd client && pnpm wasm` — build wasm (re-run after Rust changes)
2. `pnpm dev` — start Vite dev server with HMR for TS changes
3. Or `pnpm wasm:dev` — both in one command

## 4. What This Does NOT Include

- No multi-hex tiling — single hex only
- No persistence or server interaction
- No production deployment config
- No Rust-side tests for wasm bindings (tested via the client)
